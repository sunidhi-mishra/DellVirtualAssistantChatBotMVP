Objective: Build an interactive Salesforce Service Console prototype optimized for handling exclusively Out-of-Warranty (OOW) Dell systems. This specification defines layout controls, automated SLA alerts for missed chats, and dual-mode data switching for the agent workspace.1. View Layout & Screen Division (1/3 vs 2/3 Split)A. Left Panel: Unified Chat Console (1/3 Screen Width)Session Navigation Tabs: A tab bar at the top categorized by status:🟢 Active: Labeled with a Service Tag (e.g., Tag: 9XYZ789). Selecting this loads its conversation, asset details, and case logs.⚠️ Incoming: A flashing alert tab labeled New Request - Tag: 4ABC123 with a live 30-second countdown timer ticking down (e.g., Accept within 00:24s).延长 Ended: Muted tabs representing historical sessions (e.g., Closed - Tag: 2LMN456).Active Chat Stream: Displays when the 🟢 Active tab is selected.Historical Bot Transcript: Shows the initial customer routing path (Greeting $\rightarrow$ Service Tag entry $\rightarrow$ Live Agent request).Automated Agent Greeting:"Thank you for contacting Dell Technologies. My name is Arjun. I see you're reaching out about an issue with your system (Service Tag: {Service_Tag}). Let me review the details you provided, and I will help you resolve this right away. Meanwhile, please feel free to share any further details or specific error messages about the issue you are facing."B. Right Panel: Dual-Mode CRM Workspace (2/3 Screen Width)This panel features two sub-tabs at its top, allowing the agent to toggle the view between "System Details" and "Case Management".Mode Tab A: System Details (Default View)Header Bar / Live Metrics: Contains an incrementing counter (AHT: 03:45) and queue volume (📥 2 chats waiting).Component 1: Customer Profile Card: Name, Mobile Number, Alternate Mobile Number (Optional), and Email Address.Component 2: System & Hardware Details: Model (e.g., Inspiron 15), CPU, RAM, Storage, and Asset Age (Strictly constrained: e.g., Asset Age: 3 Years, 2 Months or 2 Years, 11 Months—all records must be under 4 years).Component 3: Warranty & Entitlement Status (No Universal Elements):Note: There is no global app banner indicating OOW. Instead, this card explicitly shows the status for the selected asset:Status Badge: 🔴 OUT OF WARRANTYHistorical Log: Previous Plan: ProSupport Plus | Expired: 6 Months Ago.Mode Tab B: Case Management & LogsCase Details: Case ID, Creation Date, Origin (Chat), and Current Status (Open).Interactive Case Logs: A vertical timeline of activities, automated system stamps, and notes added during the lifecycle of this Service Tag interaction.2. Interactive Behavior & State MachineState 1: Active Chat & Workspace TogglingThe agent is focused on the 🟢 Active tab.Trigger: Agent clicks between the "System Details" sub-tab and the "Case Management" sub-tab in the right 2/3 panel.Action: The UI seamlessly switches the view to show either the hardware configurations/individual warranty metrics or the running case log timeline.State 2: Accepting the Incoming Request (Interactive SLA)While the agent works, the ⚠️ Incoming tab flashes with its 30-second countdown.Trigger: Agent clicks the ⚠️ Incoming tab or hits its native "Accept Chat" button before the timer hits 00:00.Action: The tab state changes to 🟢 Active, the 30-second timer clears, the automated greeting sends, and the right panel updates with the new asset's details (e.g., Asset Age: 1 Year, 8 Months; Warranty: 🔴 Out of Warranty).State 3: Missed Request & Automated Case Logging (Timer Timeout)Trigger: The 30-second countdown timer on the incoming request hits 00:00 without being accepted.Action: The incoming session window closes (rerouted), and the system automatically generates a background event. If the agent views the Case Management logs for that routing instance, a new automated red-flagged system entry is appended:[SYSTEM ALERT - 01:08 AM]: Chat request route timed out. Agent failed to accept session within 30-second SLA limit. Session rerouted to queue.3. Visual & UX Guidelines for AntigravityAsset Consistency: Ensure all mock data fields calculated for "Asset Age" across active, incoming, or ended screens display numbers between 1.0 and 3.9 years to fit the cohort profile.Granular Warranty Indicators: Keep the 🔴 OUT OF WARRANTY alerts isolated strictly inside the individual warranty component boxes to preserve the standard Salesforce CRM layout realism.
